基本问题描述

因为 GitHub 网络不稳定问题，在使用 Gridea 同步进行上传时会存在上传失败情况

解决方案

GitHub Pages +Grideaas 功能简介

简单理解，GitHub Pages 即为每一个 GitHub 用户提供一个免费的网址，该网址使用 GitHub Pages 对应库中的资源进行生成页面。在博客制作过程中 Gridea 扮演了将用户制作的 Markdown文档渲染成 GitHub Pages 所需资源的角色。

Gridea 预览及同步功能

从 Gridea 功能出发，查阅 Gridea 项目文档 发现，Gridea 在 0.9.3 版本更新中对博客本地存储结构做了改变。

尝试发现，Gridea 中预览功能会将博客 Markdown 进行渲染，渲染后文件存储在文件夹～/.gridea/output 中，该文件夹中内容即为 GitHub Pages 所需资源。

而同步功能则会在渲染基础上，根据系统预设远程链接信息重新创建 Git 库，同时将其整合成一个 commit 进行提交。

由于同步功能因为网络不稳定问题会存在失败（同时提交时会覆盖此前 commit 记录），故考虑不使用同步功能，仅将 Gridea 客户端作为一个静态渲染器（预览功能），再渲染完成后采用 Git 进行手动提交。
